# OrderFlow

Тестовое задание. Система импорта заказов из JSON в RetailCRM, синхронизации с Supabase, визуализации данных и Telegram-уведомлений.

---

## Файлы проекта

| Файл | Описание |
|------|----------|
| `orders/mock_orders.json` | Тестовые данные заказов: покупатели, товары, адреса доставки, UTM-метки |
| `import_to_retailcrm.py` | Импортирует заказы из `orders/` в RetailCRM через API v5. Сканирует все JSON-файлы в папке автоматически |
| `sync_to_supabase.py` | Инкрементальная синхронизация заказов из RetailCRM в Supabase. При каждом запуске забирает только новые и изменённые заказы |
| `tg_bot.py` | Flask-сервер, принимает вебхуки от RetailCRM и отправляет уведомления в Telegram о заказах на сумму свыше 50 000 ₸ |
| `index.html` | Одностраничный дашборд: подключается к Supabase напрямую из браузера, отображает статистику по заказам, товарам, городам и источникам трафика |
| `schema.sql` | SQL-схема для создания таблиц `orders` и `order_items` в Supabase |
| `.env.example` | Шаблон переменных окружения |
| `requirements.txt` | Зависимости Python |

---

## Запуск

```bash
cp .env.example .env
# заполнить .env своими данными

pip install -r requirements.txt

# 1. Импорт заказов в RetailCRM
python import_to_retailcrm.py

# 2. Синхронизация в Supabase (предварительно выполнить schema.sql)
python sync_to_supabase.py

# 3. Telegram-бот (деплой на Render)
gunicorn tg_bot:app
```

**Промпт**

```

Создай дашборд index.html – единый HTML-файл для отображения данных заказов из Supabase. Файл планируется деплоить в Vercel.

Блок 1:
```

### 4. tg_bot.py

**Промпт**

```
#Задача
Напиши скрипт tg_bot.py. Данный скрипт необходим для отправки уведомлений из RetailCRM в Телеграм в случае, когда стоимость нового заказа более 50000 тенге. Существующим функционалом RetailCRM это не сделать, поскольку на текущий момент Телеграм заблокирован на территории РФ и недоступен с российских IP-адресов. Скрипт планируется размещать через сервис render.com.

#Логика работы скрипта
1)	RetailCRM отправляет POST запрос на эндпоинт скрипта /webhook с данными заказа (триггер уже настроен на стороне CRM - срабатывает только на заказы стоимостью более 50000 тенге) 
2)	Скрипт принимает запрос и извлекает номер заказа и сумму 
3)	Скрипт отправляет сообщение в Telegram через Bot API с номером заказа и суммой

#Дополнительно
Обнови файлы .env.example и, при необходимости, requirements.txt
Если у тебя есть дополнительные вопросы – перед выполнением задачи задай их.
```

**Результат**

> Скрипт был успешно реализован. 

**Сделанные доработки**

> Был добавлен комментарий в .env.example касаемо переменных окружения - они не должны будут фигурировать в итоговом варианте, будут настраиваться через интерфейс Render.

**Возникшие затруднения**

> Изначально была идея воспользоваться встроенным инструментом RetailCRM (триггер + отправка сообщения в Телеграм), однако это решение оказалось нерабочим. В процессе поиска проблем я пробовала отправить после срабатывания триггера запрос на https://webhook.site/ - он успешно прошел, из чего я сделала вывод что проблема непосредственно связана с доступом к Телеграм на территории РФ. Для полного выполнения задания я приняла решение воспользоваться сервисом Render.
